# Git and GitHub Activity

## Goal

Complete one full local-to-remote workflow using a toy repository.

## Tasks

1. Create a new folder for a toy project.
2. Initialize a Git repository.
3. Add one small text or markdown file.
4. Stage and commit the file.
5. Create a GitHub repository.
6. Push your local repository to GitHub.
7. Make one small change.
8. Pull that change back to your machine if needed.

## Questions to Keep in Mind

- What changed between your working directory and your last commit?
- What changed between your local repository and the remote repository?
- Why is this workflow useful in a restricted data setting?

## Success Check

You should end with:

- at least one local commit
- a remote GitHub repository
- confirmation that you can push and pull
